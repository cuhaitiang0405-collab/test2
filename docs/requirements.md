# 多学科会诊中心系统 — 产品需求规格（PRD / 需求追溯）

> **本文档定位**：把 `多学科会诊中心系统.docx` 中收集的 26 条功能模块，提炼为结构化需求清单，并逐条映射到 **6 大 DDD 领域** 与 **M1–M6 里程碑**，标注「是否已被现有文档记录」与「实际实现状态」。
>
> **为什么需要它**：现有 `docs/architecture.md`（架构设计）与 `docs/milestones.md`（里程碑计划）只覆盖到「领域/模块级」的高层描述，**没有一份把 docx 逐条功能点追溯到系统落地状态的文档**——这就是此前产物里「缺少产品需求记忆」的根因。本文档是需求与实现的单一事实来源（SSOT）。
