## 每里程碑统一自测清单（"字段能否更新 / 后端能否对接"）

1. **字段更新**：经前端/API 改一条记录 → 落库 → 回读一致（断言相等）。
2. **后端对接**：网关 REST → 域内 gRPC 调用返回预期（含 TraceId 透传断言）。
3. **合规**：写一条含姓名/身份证的测试日志 → 断言输出仅剩 PatientID+检查号掩码。
4. **预览**：启动 dev 环境，给出可访问 URL / 截图，标注本里程碑完成的功能。

## 假设项（如有误请纠正，否则按此推进）

- **认证**：M1 先用简单 **JWT（用户名/密码 + mock 用户表）**，后续可接医院统一认证（OAuth2）。
- **域间通信**：六域间 **gRPC**，对外统一 **REST via API 网关**（与 `proto/mdt.proto` 对齐）。
- **负载均衡**：作为 ⑥ 扩展层策略，在 **M6** 接入云影像 SPI；M1 不含。
- **真实系统**：全程 **Mock**；真实 PACS/HIS 接入作为 M2 之后的独立任务，不阻塞里程碑。
- **Impeccable**：在 **M1 的 UI（登录+空壳）** 即启用，生成医疗蓝设计 token（主色/间距/组件），后续里程碑复用。
