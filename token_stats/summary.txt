2. **Key Technical Concepts:**
   - Spring Boot 3.2.5 multi-module microservices (parent `mdt-parent` + 8 `mdt-*` modules); DDD (`XxxService`/`XxxRepository`/`XxxPort`). Java 20.
   - gRPC (net.devh grpc-spring-boot-starter 3.1.0.RELEASE) internally, REST via Spring Cloud Gateway externally.
   - Full-chain TraceId: Gateway `GlobalFilter` → servlet `RestTraceFilter` → gRPC `GrpcTraceInterceptor` (gRPC Context) → `ClientTraceInterceptor` (REST→gRPC metadata `x-mdt-traceid`). `AuditLogger.log(tenantId, operatorId, patientVisitUid, action, detail)`.
   - Compliance: `Desensitizer.mask`, `AuditLogger`, field RBAC, unified `PatientVisitUID` (`PV-{tenant}-{pid}-{acc}`).
   - M3 stack (locked): **self-built lightweight WebGL2 volume renderer** (no Cornerstone3D yet, interface seam reserved for production swap); **procedurally synthesized volume data** generated by `mdt-image` and streamed to frontend; scope = **MPR three orthogonal planes + window/level + zoom/pan + series comparison** (3D MIP optional bonus); **new real `mdt-image` microservice** (port 8083, gateway `/api/image/**`, dev phase REST-only, gRPC stub reserved); **M2 `STUDY_INDEX` list as reader entry**, top shows desensitized PatientID+AccessionNumber, `patientVisitUid` threaded through, compare two studies under same patient.
   - WebGL2 3D texture (`TEXTURE_3D`) volume rendering, fragment-shader MPR reconstruction, GPU-side windowing — off main thread.
   - Tech-debt policy: "don't dig pits" — no new dev code may directly depend on `Mock*` classes/`mock_*` table names; `IntegrationService.ingest(IngestCommand)` must stay source-agnostic.
   - Postgres 15 in Docker (localhost:5432, db `mdt`, user `mdt_app`/`mdt_pass`).
